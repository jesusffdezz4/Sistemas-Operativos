Funciones del SO

Gestión de procesos

Un proceso se crea cada vez que se ejecuta un programa, al cargar las instrucciones del mismo en memoria principal. Cada proceso necesita recursos, y el SO es el encargado de decidir qué proceso se ejecuta en cada momento, y cuál debe esperar. Por eso los procesos tienen varios estados:

  • Listo o Ready (o en espera o listo): Es el proceso para el cual no se están ejecutando órdenes, pero que está disponible para que el sistema pueda asignar y ejecutar un procesador.
  • En ejecución: Es el proceso que tiene asignado un procesador y desde el que se están ejecutando instrucciones. Puede cambiar por:
  • Bloqueado: El proceso está a la espera de un recurso ocupado y no disponible y no podrá abandonar este estado hasta que el recurso esté libre, en ese caso irá al estado Ready.
  • Creado: Se ha ordenado la ejecución del proceso y se está creando la estructura de datos necesaria y la memoria asignada. Cuando se realizan todos estos preliminares, pasa al estado Ready.
  • Zombie: En UNIX/Linux, se trata de un proceso finalizado que no se pudo eliminar por algún motivo y todavía está en la cola de procesos.
  • Suspendido: Es un proceso detenido a petición del usuario que durante el tiempo que permanece en este estado no lo selecciona el algoritmo de programación para pasar a la ejecución. Cuando se reactiva, va a Ready.

Gestión de la memoria

El SO utiliza distintas técnicas para optimizar el uso de la memoria interna, y así dar la sensación al usuario de que se ejecutan varios procesos a la vez, lo cual solo es posible si disponemos de varios procesadores, y optimizar el tiempo de respuesta de los mismos en función de su prioridad.

Por otro lado, el SO gestiona la memoria secundaria facilitando técnicas de particionado que permiten crear espacios de memoria independientes, con dos objetivos fundamentales: pueden ser gestionados aisladamente (copias de seguridad, alojar distintos SO, tener distintos sistemas de archivos, etc.) y permiten optimizar el uso de la memoria secundaria.

Gestión de entrada / salida

El SO es el encargado de gestionar la interacción con los periféricos sirviendo de intermediario e intérprete, tanto para que el sistema pueda usar la información introducida a través de ellos, como para dar el formato necesario y poder mostrar a través de los periféricos de salida, la información que el sistema quiere transmitir al exterior.

Para llevar a cabo esta función el SO emplea los drivers o controladores.

Gestión de archivos

Los usuarios quieren poder almacenar la información y luego poder recuperarla. Existen distintos tipos de dispositivos de almacenamiento pero todos ellos necesitan un sistema que les permita alojar y recuperar la información. El SO emplea los sistemas de archivos para alcanzar este objetivo. Sus funciones principales son:

  • Estructurar la información en archivos y directorios (contenedores de archivos).
  • Asignar espacio en el dispositivo a los archivos.
  • Permitir el acceso a estos archivos.
  • Gestionar el espacio libre disponible.

Gestión de la seguridad

La gestión de la seguridad implica principalmente dos aspectos diferentes:

  • El mecanismo de control de acceso, que define la forma en la que se permite o deniega a los procesos y usuarios el acceso a los diferentes recursos del sistema.
  • Control de errores que afectan el funcionamiento del sistema.

Interfaz de usuario

La interfaz es el elemento del sistema operativo a través del cual el usuario interactúa con el propio sistema. A grandes rasgos, esta interfaz debería permitir al usuario:

  • Identificarse.
  • Ejecutar programas y recibir sus resultados.
  • Gestionar su propio sistema.

Analizaremos en profundidad cada una de ellas en las próximas UD del módulo.
