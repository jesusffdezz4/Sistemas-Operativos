Sistemas operativos

En este bloque de contenidos vamos a responder a las siguientes preguntas

  • ¿Qué es un sistema operativo y cuáles son las principales funciones que realizan?
  • ¿Cuáles son los diferentes tipos de sistemas operativos que existen?
  • ¿Cómo gestionan los recursos?

El sistema operativo es el conjunto de programas informáticos básicos que se encargan de actuar como intermediario entre el hardware y el resto de aplicaciones y que permiten al usuario comunicarse con el sistema informático. Como consecuencia de que es esencial para la ejecución de las aplicaciones y la comunicación con el usuario, es el software que intenta cargar la máquina al iniciar.

Su existencia permite a las aplicaciones abstraerse de las particularidades de los componentes físicos y ejecutarse simultáneamente sin que una de ellas acapare el procesador o la memoria.

Sistema operativo monousuario, monopuesto y en red

No hay que confundir SOM -"Sistema Operativo Monopuesto" con el concepto ya obsoleto, SOM-"Sistema Operativo Monousuario":

  • SOM -"Sistema Operativo Monopuesto" Se habla de sistema operativo monopuesto cuando forma parte de una red entre iguales, está aislado o solo es cliente de servicios web. Por ejemplo, puede acceder a la web a través del navegador, o al correo electrónico, pero, a nivel de red local, se limita a comunicarse a través de la misma o como máximo compartir algún recurso, como una impresora de red o un directorio compartido y solo usa usuarios locales.
  • SOM-"Sistema Operativo Monousuario" Es un sistema operativo con un único usuario. Cuando se generalizó el uso de internet, este tipo de SO desapareció por cuestiones de seguridad. Algunos de estos sistemas podían trabajar en red local, otros no. MS-DOS, por ejemplo, no tenía capacidades de comunicación de red nativas.
  • Finalmente, se habla de SOR- "Sistema Operativo de Red" si constituye, junto con el resto de los sistemas de su red, un conjunto cooperativo de sistemas que comparten usuarios, permisos de acceso y otros recursos, generalmente porque uno o más de los sistemas actúan como servidores, mientras que el resto actúa como clientes.
